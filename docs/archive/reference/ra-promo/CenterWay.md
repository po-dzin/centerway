# CenterWay

> [!Abstract]
> CenterWay – мультипродуктовая велнес-образовательная экосистема, построенная как платформа с доменным ядром, структурированным контентом, режимным UI-генератором и несколькими ветками входа. Этот проектный индекс удерживает карту канонов, активную волну и решения без дублирования их тел.

___

## Определение

CenterWay – не просто сайт с курсами, а платформа с доменным ядром, слоем контента, обучения, практики, знаний и агентной автоматизации. Бренд и UI работают как единая экосистема маршрутов: [[Архитектура|архитектура]] держит контракты, [[Бренд-контракт|бренд-контракт]] задаёт тон и границы, [[UI-UX канон|UI-UX канон]] собирает экраны по единому закону.

___

## Продуктовые ветки

- Ветка A – платформа: `reboot.centerway.net.ua` (Short Reboot) и `irem.centerway.net.ua` (IREM Gymnastics) как самостоятельные воронки на субдоменах с единым runtime текущего репо.
- Ветка B – устаревшие страницы на WIX: Шлях21, Ідеальне Тіло, Розвантажувальний День, консультации; ручная выдача через Direct и Telegram.
- Ветка C – травы и природная поддержка как отдельная продуктовая поверхность: изолированный funnel `/herbs` + platform product page `/products/herbs`; детали физ-продукции и БАДов уточняются по мере наступления фактов.
- Доша-тест – lead-магнит и слой сегментации, общий для всех веток; спецификация в [[Доша-тест]].

Разделение маршрутов зафиксировано как непереговорное правило: никаких перекрёстных ссылок между воронками лендингов и никаких переходов на общий хаб. Детали – в [[Бренд-контракт]].

___

## Активные цели

- Удержать один проектный индекс (этот файл) и компактный набор смысловых канонов как SSoT
- Выпустить канонически укреплённый релиз текущей воронки с каноническими архетипами и primary CTA через токены (см. [[Релизный чеклист]])
- Сохранить паритет конверсии для short/irem на маршрутах R1 и нормализовать дизайн-систему через R2 (см. [[Лендинги]])
- Довести доша-тест до состояния, готового к продакшену, со сохранением попыток и аналитикой (см. [[Доша-тест]])
- Закрыть релизный гейт по безопасности, лимитам частоты и RLS до следующего релиза (см. [[Релизный чеклист]])

___

## Текущая волна

Активна волна «укрепление канона + переход платформы»:

1. Зафиксировать канонические ключи архетипов в `screen_manifests.json` (убрать слой алиасов из runtime)
2. Перенести размеры primary CTA в слой токенов (`--cw-btn-primary-width-*`) и унифицировать в hero, прайсинге, блоке следующего шага и липком CTA
3. Усилить разницу ролей для поверхностей proof/boundary/support без потери дисциплины поверхности
4. Удалить устаревшие маппинги рецептов `dosha.support.*` из реестра
5. Держать `ci:smoke` и `generator:validate/determinism/snapshot` зелёными на каждом срезе

Подробная разбивка – в [[Релизный чеклист]] и [[Мета-аудит]].

___

## Ближайший бэклог

- Канонические архетипы как SSoT в манифестах и удаление слоя алиасов в runtime
- Размеры primary CTA через токены и удаление переопределений на уровне страницы
- Токены ритма секций по фазам архетипа со снимками CI для desktop/tablet/mobile
- Доведение доша-теста до продакшена: сохранение попыток, поток напоминаний (отложено до выбора канала доставки), перезапуск
- Безопасность: глобальный лимит частоты, ревизия RLS, сканирование секретов
- Напоминания dosha через cron – отложены до выбора канала доставки и финальной модели `platform_users`

___

## Решения

- Платформа единая: текущее репо – единственный runtime; устаревшие репозитории остаются только как локальные снимки
- Токены именуются `--cw-{group}-{name}` (CSS) и `{group}.{name}` (JSON/TS); точечные переопределения CSS запрещены
- Любое сообщение, связанное со здоровьем, проходит матрицу заявлений и содержит блок границы
- Выдача доступа оперативно реализована через `access_tokens + token_consumed` до финальной ветки LMS
- Изоляция субдоменов: `reboot.*` и `irem.*` не ссылаются на `centerway.net.ua` и друг на друга
- Политика бренда и заявлений CenterWay приоритетнее паттернов UUPM

## Семантическая архитектура и runtime governance

RAverse – единый семантический и governance-авторитет для публичной архитектуры CenterWay. Репозиторий не должен порождать равноправные локальные семантические диалекты в docs, CSS, TSX или сгенерированных артефактах.

Файлы runtime в репозитории сериализуют этот канон только для исполнения:

- Data/design-tokens/cw.tokens.json
- Data/generator/screen_manifests.json
- Data/generator/block_manifests.json
- Data/generator/archetype_contracts_v0_1.json
- Data/generator/route_family_contracts.json

Цепочка публичной платформы: RAverse Canon -> runtime manifests -> generated renderer -> pages. Локальные docs остаются производными и используются только для реализации, если правило не поднято обратно в этот канон.
