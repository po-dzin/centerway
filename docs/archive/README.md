# Archive Storage

`docs/archive/` keeps useful local documentation that should remain in the repository
for provenance, but should not stay in the active root layer of `docs/`.

## Buckets

- `docs/archive/working-notes/`
  dated implementation notes, wave-specific audits, temporary design or routing work
- `docs/archive/reference/`
  extracts, derived references, and material that helps inspection but is not active policy
- `docs/archive/ops/`
  operational playbooks, analytics notes, and support artifacts that are useful historically

## Rule

If a note does not belong in the active root set from `docs/LOCAL_DOCS.md`, archive it here
unless it should instead be promoted into RAverse as durable canon.
