# План подключения нового монорепо (вместо 3 репозиториев)

## 1. Зафиксировать новый источник истины

1. Принять `platform/` как единственный runtime (API + host routing + статические лендинги).
2. Перевести CI и локальные команды на `platform/`.
3. Оставить `centerway-backend`, `irem-gym`, `short-reboot-main` только как локальные snapshots (без трекинга в git).

## 2. Подготовить репозиторий

1. Удалить старую структуру `apps/platform` из индекса.
2. Добавить новую структуру `platform/` в индекс.
3. Проверить, что в коде больше нет ссылок на `apps/platform` и `/legacy/*`.

## 3. Проверить функциональность

1. `cd platform && npm install`
2. `npm run ci:smoke`
3. Убедиться, что:
   - роутинг отдаёт контент из `/lps/{brand}/*`
   - `GET /api/health` отвечает `{ ok: true }`
   - `checkout/pay/webhook/return` проходят smoke/userflow проверки

## 4. Cutover окружений

1. Создать preview deployment новой ветки.
2. Прогнать чеклист из `docs/migration/CUTOVER_CHECKLIST.md`.
3. Переключить DNS `reboot.*` и `irem.*` на новый deployment.
4. Мониторить 48-72 часа:
   - записи `orders`, `events`, `leads`
   - успешные/неуспешные возвраты после оплаты
   - метрики конверсии по обеим воронкам

## 5. Вывод из эксплуатации старых 3 репозиториев

1. Заморозить изменения в `centerway-backend`, `irem-gym`, `short-reboot-main`.
2. Оставить только архивные теги/бэкап.
3. Рабочие изменения принимать только через этот монорепозиторий.
